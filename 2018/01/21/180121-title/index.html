    입사 3주차 후기

    기술교육 2주차

    이번주는 교육이 많았고, 바쁘게 진행되어서 그런지 금방 지나간 것 같다.
    GitHub, HTML/CSS, vi, spring, javascript 등 좋은 내용의 교육을 빠른 시간안에 들을 수 있었다. 깊고 어려운 내용은 아니었지만, 해당 분야의 공부를 시작하기엔 충분한 내용이었다.
    특히 javascript 교육 때, jQuery 등의 다른 라이브러리를 사용하지 않고 javascript만으로 DOM element를 제어하는 실습이 재미있었다. 실습 마지막 부분은 조금 놓쳤던 것 같은데, 혹시 프로젝트에 여유가 생긴다면 꼭 다시 해봐야겠다.

    루키행복TF

    이번 주의 주요 과제는 업무 세분화, 기능 분배, ERD 작성 및 제출이었다. 지난 주에 기획했던 내용들을 개발하기 위해서 설계하는 과정이었다.
    Groupware의 요구사항을 만족하고, 기간 내 개발할 수 있으면서, 모두가 공평하게 프론트부터 백엔드까지 업무를 맡을 수 있도록 분배하는 것이 제일 어려웠던 것 같다.
    지난주에는 회의 시간이 조금 오래 걸리기도 했지만, 이번주에는 실제 개발에 들어가기 직전이어서 그런지 의견이 잘 모여서 회의가 상대적으로 짧게 끝났다. 또한 다들 적극적으로 참여하다보니 자연스럽게 자기가 잘할 수 있는 일을 찾아서 하고, 그래서 TF가 더 잘 굴러가는 것 같다.

    책읽기 x TDD

    토비의 스프링 1,2장을 읽어오는 것이 주말동안 과제였는데, 책을 읽다가 프로그래밍 뿐 아니라 책읽기 과제도 TDD 방식으로 진행하면 재밌을 것 같다는 생각이 들었다.
    해당 챕터를 잘 이해했는지 확인 할 수 있는 질문을 테스트케이스처럼 작성하고, 질문에 대한 대답을 잘 해낼 수 있으면 다음 질문으로 넘어가는 것이다.

    Spring

    기획안 작성, 설계를 하면서는 ‘빨리 개발하고싶다’는 생각을 하곤했는데, 막상 개발을 시작하게 되니 걱정반 기대반의 마음이다.
    욕심 내지말고 기획, 설계해온대로만 무사히 완성시키면 좋겠다!!
